• Multiple Spanning Tree Protocol (MSTP), initially defined in IEEE 802.1s and later included in IEEE 802.1Q, supports mapping of multiple VLANs onto a single spanning-tree instance. • This reduces the number of spanning-tree instances required in a switched network with many VLANs. • Although RSTP provides faster convergence time than STP does, it still does not solve a problem inherent in STP: all VLANs within a LAN must share the same spanning tree. ######################## #Step 1 : Topology # Step 2: configure the switches: #Switch D1: enable configure terminal hostname D1 spanning-tree mode rapid-pvst line con 0 exec-timeout 0 0 logging synchronous exit int range gi0/0-2 switchport trunk encapsulation dot1q switchport mode trunk no shut exit vlan 2 name SecondVLAN exit vlan 3 name ThirdVLAN exit vlan 4 name FourthVLAN exit vlan 5 name FifthVLAN exit end wr ####################### # Switch D2: enable configure terminal hostname D2 spanning-tree mode rapid-pvst line con 0 exec-timeout 0 0 logging synchronous exit int range gi0/0-2 switchport trunk encapsulation dot1q switchport mode trunk no shut exit vlan 2 name SecondVLAN exit vlan 3 name ThirdVLAN exit vlan 4 name FourthVLAN exit vlan 5 name FifthVLAN exit end ##################### # Switch A1: enable configure terminal hostname A1 spanning-tree mode rapid-pvst line con 0 exec-timeout 0 0 logging synchronous exit int range gi0/0-3 switchport trunk encapsulation dot1q switchport mode trunk no shut exit int gi1/0 no shut exit vlan 2 name SecondVLAN exit vlan 3 name ThirdVLAN exit vlan 4 name FourthVLAN exit vlan 5 name FifthVLAN exit end wr ################################# # Step 3: Implement and Observe MST.
